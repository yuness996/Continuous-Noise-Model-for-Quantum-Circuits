# Random Clifford Circuits

This folder tests the analytical approximation of the continuous-noise model on random Clifford circuits.

For each random circuit, the code compares:

* the **full model**, where a fresh coherent error is inserted after each noisy single-qubit gate;
* the **approximate model**, where the accumulated local error is replaced by one effective error at the end of each qubit.

The comparison metric is

```text
infidelity of approximate model
--------------------------------
    infidelity of full model
```

A value close to `1` means that the approximation reproduces the full-model infidelity well.

## Files

```text
Random Clifford Circuits/
├── RandCirc.py
└── RetrieveInfo.py
```

* `RandCirc.py` contains the original simulation.
* `RetrieveInfo.py` currently contains a cleaned and commented version of the same simulation.

> **Note:** `RetrieveInfo.py` does not currently retrieve data from an output file. Rename it or replace it with a post-processing script to avoid confusion.

## Simulation method

The code generates random circuits containing fixed numbers of:

* Hadamard gates;
* CNOT gates.

At least one Hadamard gate is placed on each qubit, and the full gate list is randomly shuffled.

The continuous error applied after a noisy single-qubit gate is

```text
Z Rx(2 theta) Rz(-2 phi) Z
```

with Gaussian angles of standard deviation `sigma`.

In the approximate model, local error variances are propagated analytically. CNOT gates add no fresh noise, and correlated errors generated by their propagation are ignored.

The ideal, full-noise, and approximate circuits are simulated with Qiskit Aer. Their final statevectors are compared using

```text
F = |<psi_ideal | psi_noisy>|².
```

## Output

The default run evaluates a (10\times10) grid:

```text
Rows:    n_cx = 10, 20, ..., 100
Columns: n_h  = 10, 20, ..., 100
```

Each entry contains

```text
infidelity_approx / infidelity_full.
```

The matrix is appended to:

```text
RandomCircuits10qbts_new.txt
```

Each run adds ten new rows. Delete or rename the old file before starting a separate experiment.

The simulation can be parallelized by running independent jobs with different random seeds. It is safer for each job to write to a separate file:

```text
RandomCircuits10qbts_job_001.txt
RandomCircuits10qbts_job_002.txt
...
```

The files can then be combined or averaged during post-processing.
